Projektbeschreibung

Places ist eine Web Anwendung, welche vom grundlegenden Design an Airbnb o.ä. erinnert. Das Ziel ist es, alle potenziell interessanten Locations in einer App zu vereinen und mit einem Suchalgorithmus zugänglich zu machen. Dies erreichen wir, indem wir mit der App alle interessanten Drehorte in einem Interface darstellen und sie mit Tags versehen. Die Tags helfen dann dabei, mit einem Filter nur die Orte herauszusuchen, welche für den jeweiligen Dreh oder das Shooting passend sind. Neben den Filtern gibt es zahlreiche weitere Funktionen, um z.B. die Adresse, den Preis etc. einzusehen oder eigene Orte hochzuladen.

Problem

Jede content-schaffende Person im Bereich der Musikvideos benötigt für eigentlich jedes Projekt drei Dinge:

  • Das Equipment
  • Den Künstler
  • ≥ 1 Drehort

Vor allem der dritte Punkt gestaltet sich gerade für kleine Produktionen häufig am schwierigsten, da für viele beliebte Drehorte eine Drehgenehmigung erforderlich ist, welche wiederum hohe Kosten verursacht, oder ein extremer Zeitaufwand für das Scouting der Locations nötig ist, bis man etwas einigermaßen Passendes gefunden hat.

Aus eigener Erfahrung können wir sagen, dass es oft darauf hinausläuft, dass man sich stundenlang durch die Straßen Berlins und Umgebung begibt, um einen Ort zu finden, welcher zum Stil des Videos passt.

Value Proposition

Zielgruppe:

  • Kleine Video- und Filmproduktionen
  • Fotografen
  • Hobby-Content-Creator

Geographischer Fokus:

Zunächst auf den Raum Berlin beschränkt

Places bietet eine zielgerichtete und einfache Suche nach geeigneten Drehorten. Das Finden des perfekten Drehortes kann ein zeitaufwändiger und kostspieliger Prozess sein, insbesondere wenn viele beliebte Orte teure Drehgenehmigungen erfordern. Unsere Plattform vereinfacht das Location-Scouting durch:

  • 1. Umfassende Datenbank: Zugriff auf eine Vielzahl potenzieller Dreh- und Fotolocations, die speziell auf Ihre Bedürfnisse zugeschnitten sind.
  • 2. Benutzerfreundliche Suche: Filtern Sie Orte nach spezifischen Kriterien wie Szenerie, Ausstattung und Zugänglichkeit, um schnell den perfekten Ort zu finden.
  • 3. Kostengünstige Lösungen: Entdecken Sie weniger bekannte Orte, die keine teuren Genehmigungen erfordern, und bleiben Sie im Budget.
  • 4. Zeitersparnis: Verbringen Sie weniger Zeit mit der Suche nach Locations und mehr Zeit mit der Erstellung von Inhalten dank unseres optimierten Scouting-Prozesses.
  • Lokale Einblicke: Profitieren Sie von detaillierten Informationen und Bewertungen zu jedem Ort, die von anderen Kreativen und lokalen Experten bereitgestellt werden.

Indem sich Places zunächst auf Berlin konzentriert, streben wir an, die bevorzugte Ressource für Content Creator zu werden, die den perfekten Hintergrund für ihre Projekte suchen und den Prozess der Location-Suche so effizient und kostengünstig wie möglich gestalten möchten.

MVP

  • Einfachen Zugang ermöglichen

App Architecture

Client-Side

  • JavaScript: Used for DOM manipulation, event handling, and making asynchronous HTTP requests.
  • HTML: Provides the structure for the pages, including modal dialogs and forms for login and registration.
  • CSS: Styles the elements, including the filter bar, buttons, dialogs, and map.
  • Mapbox GL JS: For interactive maps, user location visualization, and geographical features.
  • Firebase: For authentication and real-time database functionalities.

Server-Side

  • Flask: (Assumed from the template tags and structure) For handling routing, rendering templates, and serving static files.
  • Firebase: Backend services for storing user data, managing authentication, and synchronizing user actions with the database.

Design Decisions/h2>

Client-Side

  • Event Listeners: Used extensively to handle user interactions such as clicks, input changes, and DOM content loading.
  • Fetch API: Used for making HTTP requests to the server, ensuring a seamless user experience with asynchronous operations.
  • Geolocation: Utilizes the browser’s geolocation API to get and store user’s location, enhancing the app’s personalization features.
  • Mapbox Integration: Provides a powerful and interactive map interface for users to visualize locations and apply geographical filters.

UI Components

  • Filter Bar: Allows users to filter locations based on various criteria (geographical radius, type, theme, use case). Each filter button triggers a dialog for detailed selection.
  • Modals: Used for login, registration, and displaying detailed location previews with images. This enhances user experience by providing more information without navigating away from the main page.
  • Carousel: For displaying multiple images within location previews in a sleek and user-friendly manner.

Firebase

  • Authentication: Manages user login and registration, ensuring secure access to personalized features.
  • Real-Time Database: Stores user data, such as profile information and location preferences, allowing real-time synchronization across devices.

Key Solutions/h2>

User Authentication and Data Management

  • Firebase Auth and Database Integration: Ensures that user data is securely stored and easily accessible. User actions such as login, registration, and last login updates are seamlessly handled.
  • Error Handling: Alerts users to fill out all fields and displays error messages from Firebase to guide user actions.

Geolocation and Map Visualization

  • Mapbox Integration: Offers a rich interactive map experience, with features like user location markers, adjustable radius circles for filtering, and dynamic map resizing.
  • Geolocation Handling: Saves and retrieves user location from localStorage, enabling personalized map views and filters.

UI/UX Enhancements

  • Dynamic Dialogs: Modular dialogs triggered by filter buttons ensure a clean and organized interface. They are positioned relative to their triggers and can be closed easily.
  • Interactive Elements: Sliders for adjusting radius, carousels for image previews, and responsive buttons enhance the interactivity and user engagement of the app.

Responsive Design

  • CSS Styling: Ensures that the interface is user-friendly across different devices, with components like filter bars, buttons, and carousels being styled for responsiveness and ease of use.

Modular Code Structure

  • Separation of Concerns: JavaScript functions are modular, focusing on specific tasks like map initialization, request handling, and UI updates. This enhances code maintainability and readability.
