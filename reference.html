Startseite

Abschnitt / Modul: Hauptansicht

Route: /

Methoden: GET, POST

Zweck: Die Startseite zeigt eine Liste von Orten an, die nach den vom Benutzer ausgewählten Tags und dem maximalen Entfernungsradius gefiltert werden. Bei POST-Anfragen werden Filterkriterien wie Tags und Entfernung aktualisiert.

Beispielausgabe:

GET /: Gibt die HTML-Seite zurück, die die gefilterten Orte und deren Bilder anzeigt. Die Tags werden basierend auf den Benutzereinstellungen und den ausgewählten Filtern gefiltert.

Tags Empfang

Abschnitt / Modul: Tag-Verarbeitung

Route: /recieveTags

Methoden: POST

Zweck: Diese Route empfängt Filterkriterien wie Tags und maximale Entfernung vom Benutzer und aktualisiert die globalen Filtereinstellungen.

Beispielausgabe:

POST /recieveTags: {"tags": ["Park", "Museum"], "max_distance": 5, "user_lat": 52.52, "user_lon": 13.405}

Login

Abschnitt / Modul: Authentifizierung

Route: /login

Methoden: GET, POST

Zweck: Die Login-Seite zeigt ein Login-Formular an. Bei POST-Anfragen wird der Benutzer authentifiziert.

Beispielausgabe:

GET /login: Gibt die HTML-Seite mit dem Login-Formular zurück.

Registrierung

Abschnitt / Modul: Authentifizierung

Route: /registration

Methoden: GET, POST

Zweck: Die Registrierungsseite zeigt ein Registrierungsformular an. Bei POST-Anfragen wird ein neuer Benutzer erstellt.

Beispielausgabe:

GET /registration: Gibt die HTML-Seite mit dem Registrierungsformular zurück.

Favoriten

Abschnitt / Modul: Favoritenverwaltung

Route: /favorites

Methoden: GET

Zweck: Zeigt die Favoriten des angemeldeten Benutzers an. Wenn der Benutzer nicht angemeldet ist, wird er auf die Login-Seite weitergeleitet.

Beispielausgabe:

GET /favorites: Gibt die HTML-Seite zurück, die die Favoriten des Benutzers anzeigt. Der Benutzer muss angemeldet sein.

Benutzer erstellen

Abschnitt / Modul: Authentifizierung

Route: /create_user

Methoden: POST

Zweck: Diese Route erstellt einen neuen Benutzer in Firebase Auth und speichert die Benutzerdaten in der Firebase-Datenbank.

Beispielausgabe:

POST /create_user: {"email": "user@example.com", "password": "securepassword", "full_name": "John Doe"}

Benutzer-Login

Abschnitt / Modul: Authentifizierung

Route: /login_user

Methoden: POST

Zweck: Diese Route authentifiziert den Benutzer und speichert die Benutzerdaten in der Session.

Beispielausgabe:

POST /login_user: {"email": "user@example.com", "password": "securepassword"}

Logout

Abschnitt / Modul: Authentifizierung

Route: /logout

Methoden: GET

Zweck: Meldet den Benutzer ab, indem die Session gelöscht wird, und leitet ihn zurück zur Startseite.

Beispielausgabe:

GET /logout: Leitet den Benutzer zur Startseite weiter und löscht die Session-Daten.

Favorit hinzufügen

Abschnitt / Modul: Favoritenverwaltung

Route: /set_favorite

Methoden: POST

Zweck: Diese Route fügt eine Location zur Liste der Favoriten des Benutzers hinzu. Der Benutzer muss angemeldet sein.

Beispielausgabe:

POST /set_favorite: {"favorite": "location_id"}

Favorit entfernen

Abschnitt / Modul: Favoritenverwaltung

Route: /delete_favorite

Methoden: POST

Zweck: Diese Route entfernt eine Location aus der Liste der Favoriten des Benutzers. Der Benutzer muss angemeldet sein.

Beispielausgabe:

POST /delete_favorite: {"favorite": "location_id"}
